ARBETSPROCESS

Uppstart

Projektet började med att jag skapade en mapp för själva projektet. Jag öppnade terminalen och initierade ett nytt NPM-projekt genom att ange kommandot npm init.

Jag öppnade sedan mappen i VSC och tog bort main samt script i package.json-filen. I VSC's terminal installerade parcelpaketet som ett utvecklingsberoende genom att skriva in kommandot npm install parcel --save-dev.

I package.json-filen lade jag till source för att välja vilken fil som projektet ska starta på, vilket sattes till src/index.html. Jag lade även till scripts som objekt med start och build. Genom kommandot npm run start kan jag köra webbplatsen "live" och se ändringar jag gör samtidigt som jag utvecklar. Genom kommandot npm run build paketeras projektet istället för publicering.

Jag skapade src-mappen och lade till alla mina undermappar och filer här. Mapparna består av en mapp för CSS, en för JavaScript samt en för bilder. Direkt i src-mappen finns de tre html-sidorna till webbplatsen och en favicon i ico/png-fomrmat.

I html-filernas head lade jag till "type=module" i sökvägen till min script-fil. Detta för att js-optimeringen ska ske korrekt från parcel.

Git-repository

När mitt npm-projekt var färdiguppstartat, körde jag kommandot git init för att skapa ett nytt git repository. Jag skapade även en gitignore-fil där jag valde att lägga in mapparna node_modules, dist samt parcel-cache för att inte versionshantera dessa.

Till projektet använde jag två branches, en för utveckling (development) och en för publicering (main). Under utvecklingsfasen gjordes commits till development-grenen. När projektet väl publicerades gjordes istället en merge från development-grenen till main-grenen. Samtliga commits pushades direkt till mitt remote repository på GitHub.

Utveckling

Utvecklingen av projektet har skett genom att jag har kört kommandot npm run start för att kunna se mina ändringar i realtid och även se hur parcel har minfierat och optimerat filer.

På bild-sidan har alla bilder lagts in i picture-taggen för att visa dessa beroende på besökarens skärmbredd samt webbläsarens stöd. Bilderna har fått parametrarna "as" och "width", där "as" står för vilket filformat som bilden ska konverteras till och "width" för vilken bredd som bilden ska omskalas till. Exempelvis: "?as=avif&width=800".

Publicering

När projektet var redo för publicering körde jag kommandot npm run build för att paketera och optimera mina filer inför publieringen. Projektet har sedan publiceras via Netlify.

På Netlify valde jag att lägga till en ny sida genom att importera ett existerande projekt. Jag valde "deploy with GitHub" och valde mitt git-repository till projektet. I inställningarna valde jag att main-branchen skulle överföras, dvs. min publicerings-branch. Build-kommandot är satt till npm run build vilket stämmer överrens med det kommando jag använder för att göra filerna redo för publicering. Den katalog som används för publicering är dist-katalogen.
